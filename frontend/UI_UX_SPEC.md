## Общая архитектура приложения

Приложение — одностраничное. Три экрана переключаются между собой без перезагрузки страницы. При переходе между экранами контент появляется с плавной анимацией снизу. Максимальная ширина ограничена шириной телефона — приложение предназначено исключительно для мобильного устройства (Android).

**Навигация:**

- Дашборд → Экран цеха: тап по карточке цеха
- Экран цеха → Детали аппарата: тап по карточке аппарата
- Детали → назад в цех: кнопка «назад» в шапке
- Экран цеха → назад на дашборд: кнопка «назад» в шапке
- На кнопку «назад» нет жеста свайпа — только тап

## Системное цветовое кодирование статусов

Цвет — единственная и консистентная сигнальная система во всём приложении. Нельзя использовать эти три цвета для любых других смыслов:

| Цвет    | Смысл                                               | Поведение границы карточки |
| ------- | --------------------------------------------------- | -------------------------- |
| Зелёный | Всё в норме, аппарат/цех работает штатно            | Статичная граница          |
| Жёлтый  | Есть событие, требующее внимания, но не блокирующее | Граница пульсирует         |
| Красный | Критическая ситуация, блокирующая дальнейшую работу | Граница пульсирует         |

Оранжевый цвет зарезервирован **исключительно** для FAB-кнопки «Последняя партия» — это цвет действия/оповещения, он не несёт статусного смысла и не пересекается с системой выше.

## Экран 1: Дашборд (список цехов)

### Шапка

Слева — верхняя строка мелким текстом с подписью площадки («Площадка г. Брест»), под ней крупно название предприятия («Савушкин продукт»). Текст только информационный, не кликабельный.

Справа вверху — круглая кнопка-иконка настроек (⚙️). При нажатии открывается нижняя шторка настроек. Кнопка всегда видима на этом экране.

### Список цехов

Каждый цех — карточка на всю ширину. Карточка кликабельна целиком (тап → переход на экран цеха). При нажатии карточка слегка уменьшается в размере (haptic-ощущение нажатия).

**Содержимое карточки цеха:**

- Название цеха (крупно, жирно)
- Количество аппаратов/линий всего
- Количество проблемных (если 0 и статус нормальный — подсвечивается нейтрально; если > 0 — подсвечивается в цвет проблемы)
- Бейдж статуса в правом нижнем углу карточки: «В норме» / «Внимание» / «Критично»

Цвет границы карточки — по системе статусов. Жёлтые и красные карточки пульсируют, привлекая внимание. Зелёные — статичны.

**Состояние загрузки:** пока данные не пришли, вместо карточек — текст «Загрузка цехов...» по центру.

**Состояние пустого списка:** если сервер вернул пустой массив — текст «Нет данных о цехах».

**Автообновление:** список цехов обновляется автоматически каждые 5 секунд в фоне. Никакого спиннера или индикатора обновления не показывается — данные просто тихо перерисовываются.

## Экран 2: Список аппаратов цеха

### Шапка

Кнопка «назад» (стрелка влево) — возвращает на дашборд. Рядом — название цеха, в который зашёл пользователь.

### Список аппаратов

Каждый аппарат — карточка. Карточка кликабельна целиком.

**Содержимое карточки аппарата:**

- Название аппарата/линии (крупно, жирно)
- Бейдж текущего статуса в правом верхнем углу
- Текущее событие — короткая строка курсивом (например: «Заполнение 85%», «Замятие ленты крышек»)
- Блок таймера простоя — отображается **только если статус не «Норма»**. Показывает надпись «ВРЕМЯ ПРОСТОЯ» и счётчик в формате ЧЧ:ММ:СС. Цвет блока соответствует цвету проблемы (жёлтый фон для предупреждения, красный для критического)

На карточках аппаратов нет цветовой границы как у цехов — статус виден через бейдж и (если проблема) через таймерный блок.

**Состояние загрузки:** при первом входе в цех — текст «Загрузка аппаратов...».

**Автообновление:** пока открыт этот экран — список аппаратов обновляется каждые 5 секунд.

## Экран 3: Детали аппарата

### Шапка

Фиксированная (не уходит при скролле). Слева — кнопка «назад». Рядом — подпись «Мониторинг линии» мелко и название конкретного аппарата крупно. Справа — индикатор-точка: зелёная (аппарат работает) или красная пульсирующая (есть ошибка). Индикатор обновляется при каждом опросе данных.

### Стартовая вкладка при входе

При открытии экрана смотрим: есть ли активные ошибки у аппарата?

- Если **есть** → открывается вкладка **«Журнал»** сразу
- Если **нет** → открывается вкладка **«Партия»**

Это сделано для того, чтобы в проблемной ситуации человек видел ошибки первым делом, а не тратил время на поиск вкладки.

### Нижний таббар

Фиксирован над нижней границей экрана. Учитывает системную зону навигации Android (safe-area-inset-bottom). Отображается **только** на экране деталей — на дашборде и на экране цеха его нет.

Четыре таба: Партия (📦), Устройства (⚙️), Очередь (📋), Журнал (⚠️). Активный таб выделяется — темнее и жирнее текст. Переключение мгновенное, анимация появления контента — плавный fadeIn снизу.

В перспективе: на иконке «Журнал» должен появляться красный числовой бейдж с количеством активных ошибок, если они есть. Это позволит человеку видеть ошибки, не переключая вкладки.

## Вкладка «Партия»

Одна карточка — «Текущая партия». Не кликабельна (нет hover-эффекта).

**Основные поля (видны всегда):**

- Описание (что сейчас производится, например «Сырок гл. МАКОВКА 20% 40г»)
- EAN (штрих-код продукта)
- Номер партии
- Дата выработки
- Дата годности

Каждое поле — строка: слева название поля (серый текст), справа значение (тёмный жирный текст). Строки разделены тонкими разделителями. Если данные не пришли — показывается прочерк.

**Аккордеон «Все свойства»:**
Под основными полями — кнопка «Показать все свойства ▾» на всю ширину карточки. При нажатии разворачивается блок с дополнительными полями: краткий код, КМС, дата фасовки, начальный счётчик, площадка, ITF, ёмкость, кол-во коробок, кол-во упаковок, заморозка, регион, дизайн, печать DM. Текст кнопки меняется на «Скрыть дополнительные свойства ▴». Повторное нажатие — сворачивает. Поведение независимо от FAB-кнопки: аккордеон раскрывается внутри карточки, FAB при этом остаётся на месте и никуда не уходит.

## FAB-кнопка «Последняя партия»

Плавающая кнопка, закреплённая над нижним таббаром с левой стороны экрана (небольшой отступ от левого края). Ширина — примерно половина экрана.

### Видимость

- Отображается **только** на вкладке **«Партия»**
- При переключении на любой другой таб — исчезает плавно (fade out)
- При возврате на «Партию» — появляется снова (fade in), всегда в развёрнутом виде
- При уходе с экрана деталей (нажата кнопка «назад») — немедленно скрывается

Дополнительное условие видимости (реализуется вместе с логикой «моего аппарата»):

- Кнопка показывается ТОЛЬКО если пользователь находится в экране деталей своего аппарата — того, который он указал в настройках
- Если пользователь просматривает чужой аппарат — FAB не появляется вообще, даже если открыта вкладка «Партия»

### Развёрнутое состояние (по умолчанию)

Скруглённая пилюля. Слева — иконка 🔔, рядом — текст «Последняя партия». Это состояние показывается при появлении кнопки и при скролле в верх.

### Свёрнутое состояние (при скролле вниз)

Когда пользователь скроллит вниз, текст плавно исчезает, кнопка схлопывается до круглой иконки (только 🔔). Это освобождает пространство для чтения контента. Порог срабатывания — небольшой, чтобы не реагировать на случайные микродвижения.

При скролле вверх — кнопка плавно разворачивается обратно в пилюлю с текстом.

### Нажатие на кнопку

При нажатии: визуальный фидбек — текст меняется на «Отправлено!», иконка на ✅, цвет кнопки меняется на зелёный. Через ~2 секунды кнопка возвращается в исходный вид. Это подтверждает пользователю, что сигнал ушёл.

Фактически кнопка отправляет POST-запрос бэкенду с сигналом «текущая партия — последняя». Бэкенд сам разбирается, кому по производственной цепочке нужно отправить уведомление.

## Вкладка «Устройства»

Набор карточек — по одной на каждое подключённое к аппарату устройство. Карточки не кликабельны.

**Принтер:**

- В заголовке карточки: название устройства слева, бейдж статуса справа
- Бейдж: «В работе» (зелёный), «Ошибка» (красный), «Остановлен» (серый)
- Поле: текущая партия (идентификатор того, что сейчас печатается)

**Камеры (одна или несколько):**

- В заголовке: название камеры и её роль (поток), бейдж статуса
- Поле: текущая партия
- Плитка счётчиков: два блока рядом — «Считано» (нейтральный фон) и «Несчитано» (красный фон). Крупные числа. Несчитанное — всегда красный фон, т.к. это сигнал о потенциальном дефекте.

---

## Вкладка «Очередь»

Одна карточка — «Очередь печати».

Список позиций очереди. Каждая позиция:

- Порядковый номер в кружке (слева)
- Номер партии (крупнее)
- Краткий код и дата выработки (мельче, серым)

Если очередь пуста — по центру карточки текст «Очередь пуста».

---

## Вкладка «Журнал»

Две карточки внутри вкладки.

**Карточка «Активные ошибки»:**

- Если ошибок нет — по центру карточки зелёный текст «✅ Нет активных ошибок»
- Если ошибки есть — список ошибок. Каждая ошибка оформлена блоком с красной левой полосой: название объекта (устройства) и описание ошибки. Показываются только ошибки с активным флагом (value = 1), неактивные не отображаются.

**Карточка «Журнал событий»:**

- Хронологический список событий. Каждая запись: дата и время (мелко, серым), рядом — бейдж с группой события (например «Система», «Ошибка»), ниже — описание события.
- Если журнал пуст — текст «Журнал пуст».

## Нижняя шторка настроек

Открывается кнопкой ⚙️ в правом верхнем углу дашборда. Реализована как нижняя шторка (bottom sheet), выезжающая снизу поверх текущего экрана. Закрывается свайпом вниз или нажатием на затемнённый фон за шторкой.

### Секция «Мой аппарат»

Показывает, какой аппарат сейчас выбран как «мой»:

- Если аппарат выбран: название аппарата и название цеха, к которому он относится. Кнопка «Изменить» запускает процесс смены аппарата.
- Если аппарат ещё не выбран: текст «Не выбран» и кнопка «Выбрать».

При нажатии «Выбрать» / «Изменить»: шторка переходит в режим выбора — показывается список цехов. При тапе на цех — раскрывается список аппаратов этого цеха. При тапе на аппарат — он сохраняется как «мой», шторка закрывается. Выбор хранится в localStorage устройства и никуда не передаётся.

### Секция «Тема»

Переключатель между светлой и тёмной темой. Два варианта, выбрать можно только один. Переключение применяется мгновенно без перезагрузки. Выбор сохраняется в localStorage.

## Логика «Моего аппарата» (без авторизации)

Философия: телефон = конкретный сотрудник. На производстве у каждого физически свои устройства. Поэтому достаточно один раз указать «я работаю за этим аппаратом» — и телефон запоминает это.

**Первый запуск:**
При открытии приложения проверяется localStorage. Если запись «myUnitId» отсутствует — поверх дашборда показывается оверлей с предложением выбрать свой аппарат. Оверлей нельзя закрыть свайпом, только явным действием:

- Выбрать аппарат (сохраняется и закрывается)
- Нажать «Пропустить» (закрывается, FAB нигде не показывается)

**Логика показа FAB:**
При каждом переходе на экран деталей аппарата:

- Считываем ID текущего аппарата
- Сравниваем с «myUnitId» из localStorage
- Совпадает → FAB активируется (появляется при открытии вкладки «Партия»)
- Не совпадает или «myUnitId» не задан → FAB не показывается вообще

**Смена аппарата:**
Доступна в любой момент через настройки. Поводом может быть смена смены, перевод сотрудника на другую линию. Два тапа в настройках — и телефон привязан к новому аппарату.

## Состояния загрузки и ошибок

- **Первичная загрузка** (данных ещё нет): текст-заглушка по центру, например «Загрузка цехов...»
- **Пустой список**: информационный текст по центру, например «Нет данных»
- **Недоступность сервера**: тихий fallback на моки в dev-режиме; в prod — показывать последние успешно полученные данные с индикатором устаревания
- **Загрузка значений в карточках**: если конкретное значение ещё не пришло — прочерк «-»

---

## Поведение при нажатии на элементы

- Любая кликабельная карточка: уменьшается до ~98% размера при нажатии (scale), возвращается при отпускании
- Кнопки-иконки (назад, настройки): при нажатии фоновый круг слегка светлее
- FAB: слегка уменьшается при нажатии
- Кнопка аккордеона: фон становится чуть темнее при нажатии
- Таббар: смена активного таба не имеет анимации самого таббара — только контент перерисовывается с fadeIn

---

## Глобальные ограничения интерфейса

- Горизонтальный скролл полностью запрещён
- Весь экран занимает 100% высоты viewport, нет сдвигов при показе/скрытии клавиатуры (клавиатура в приложении не используется)
- Нет элементов, требующих ввода текста (поиск, фильтры) — только чтение
- Максимальная ширина контента ограничена шириной мобильного телефона
