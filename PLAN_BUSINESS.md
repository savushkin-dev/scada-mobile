# Мобильное приложение SCADA для цеха десертов

## Текущее состояние

- (+) Backend — принимает теги оборудования с сервера PrintSrv, восстанавливается после сбоев;
- (+) Фронтенд — визуальный прототип;
- (+) Android — собран, готов к установке как прототип;
- Ограничение — сейчас поддерживается только одна линия.

## Минимальный план

**Цель**: система работает и может быть продемонстрирована.

- Поддержка нескольких производственных линий одновременно;
- Автообновление данных без перезагрузки страницы;
- Мгновенные уведомления сотрудников о событиях на участке;
- Уведомление об аппаратной ошибке не исчезает, пока проблема не устранена — статус приходит от оборудования;
- Список оборудования с цветовым индикатором состояния (работает / стоит / ошибка), отображение информации о номере партии, дате выработки, производимом продукте, количестве маркированных единиц продукта;
- Оповещение пользователей приложения смежных участков о завершении партии: работник фасовочной линии отмечает в интерфейсе, что отправил последнюю партию — все, кто стоит дальше по цепочке, сразу об этом узнают;
- Отображение индикатора наличия связи с сервером
- Адаптация под условия цеха: смартфон, плохое освещение, перчатки; тёмная тема, вибрация при тревоге;
- Пусконаладочные работы.

## Оптимистичный план

**Цель**: система внедрена на предприятии.

- Приложение отображает работу всего фасовочного оборудования цеха десертов;
- Панель оповещения для ИТ-службы: состояние работы системы, история инцидентов;
- Развёртывание одной командой (Docker / Kubernetes);
- CI/CD: автосборка и проверки при обновлении кода;
- Инструкция для системного администратора предприятия;
- Руководство пользователя;
- Выполнение бизнес-логики с помощью функциональных блоков.
