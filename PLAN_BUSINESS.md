# Система оперативного оповещения о простоях — План работ

## Контекст

При остановке одного участка цепочки (нет расходников, ошибка оборудования, оператор не среагировал) сотрудники соседних цехов не понимают причину. Возникает «эффект домино»: встаёт упаковка, палетирование, отгрузка. Система решает эту проблему — мгновенно доносит до всех нужных сотрудников, что произошло, где и почему.

## Текущее состояние

- **Backend** — работает: принимает данные с оборудования, восстанавливается после сбоев
- **Фронтенд** — визуальный прототип
- **Android** — собран, готов к установке как прототип
- **Ограничение** — сейчас поддерживается только одна производственная линия

---

## Минимальный план

Цель: система работает и может быть продемонстрирована.

- [ ] Поддержка нескольких производственных линий одновременно
- [ ] Автообновление данных без перезагрузки страницы
- [ ] Мгновенные уведомления сотрудников о проблемах на участке
- [ ] Уведомление не исчезает, пока проблема не устранена — статус прихо­дит от оборудования, а не выставляется вручную
- [ ] Список участков с цветовым индикатором состояния (работает / стоит / ошибка)
- [ ] Возможность подать команду на участок из интерфейса
- [ ] Отображение последнего известного состояния при потере связи
- [ ] Адаптация под условия цеха: смартфон, плохое освещение, перчатки; тёмная тема, вибрация при тревоге
- [ ] Сквозная проверка: событие на оборудовании — уведомление на телефоне сотрудника

## Оптимистичный план

Цель: система в продакшене на предприятии.

- [ ] Всё из минимального плана
- [ ] Дашборд для ИТ-службы: состояние связи с оборудованием, история инцидентов
- [ ] Развёртывание одной командой (Docker / Kubernetes)
- [ ] CI/CD: автосборка и проверки при обновлении кода
- [ ] Установка на телефоны сотрудников, подключение к реальным линиям
- [ ] Инструкция для системного администратора предприятия
- [ ] Трансляция графического вида рецепта в табличный вид
