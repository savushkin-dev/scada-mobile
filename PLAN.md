# SCADA Mobile — Технический план

Актуальность: 30.03.2026.

## Коротко о текущем состоянии

Система уже собрана как рабочий каркас:

- backend на Spring Boot принимает и обрабатывает данные;
- frontend на React показывает состояние и live-обновления;
- Android-оболочка через TWA подготовлена;
- есть локальный и Docker-запуск.

Подробный статус компонентов: [README.md](README.md).

## Риски и как с ними работать

| Риск | Вероятность | Что делаем |
|---|---|---|
| Нестабильная сеть или недоступность части PrintSrv | Средняя | Держим деградационный режим и явный статус «нет данных» |
| Неполный набор prod-переменных окружения | Высокая | Проверяем env-файл до запуска и используем fail-fast |
| Шумные уведомления при нестабильных ошибках | Средняя | Настраиваем правила уведомлений и дедупликацию |
| Смещение сроков доступа к промышленной инфраструктуре | Средняя | Параллельно готовим локальную репетицию через Docker |

## Критерий готовности к защите

Проект считается готовым, когда одновременно выполняются условия:

- сценарий «инцидент на линии → уведомление пользователю» работает стабильно;
- запуск по инструкциям повторяем на чистом окружении;
- документация не противоречит коду и конфигурации;
- ключевые бизнес-цели из [PLAN_BUSINESS.md](PLAN_BUSINESS.md) закрыты.
